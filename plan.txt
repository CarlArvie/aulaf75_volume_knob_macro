# Aula F75 Keyboard Macro System Plan

## Overview
We are building a macro system that leverages the volume knob of the Aula F75 keyboard. The system has two main parts:
1. **Frontend (UI):** A C# transparent overlay that shows a circular/radial menu (similar to GTA 5 weapon wheel) when triggered.
2. **Backend (Logic):** An AutoHotkey (AHK) script that detects knob rotation, communicates with the C# UI, and executes the actual macros.

## Architecture & Flow
1. **Trigger:** The user rotates the volume knob (Volume Up / Volume Down).
2. **AHK Intercept:** The AHK script intercepts the media key event, preventing it from changing the system volume.
3. **Communication (AHK -> C#):** AHK sends a signal to the C# application to display the radial menu.
4. **Navigation:** Further rotations of the knob send "Next" or "Previous" signals to the C# UI to highlight different macro slices.
5. **Selection:** The user presses the volume knob (Mute). AHK sends a "Select" signal to C#.
6. **Hierarchy (Grand-parent -> Parent -> Child):** 
   - If the selected item is a category, the C# UI updates to show the children of that category.
   - If the selected item is a final macro, C# sends a command back to AHK containing the Macro ID.
7. **Execution (C# -> AHK):** AHK receives the Macro ID and executes the corresponding keystrokes/actions.

## Components to Build

### 1. The Configuration File (JSON)
We need a unified file where macros and categories are defined so both C# and AHK can understand the hierarchy.
Example structure:
- Media Controls (Category)
  - Play/Pause (Macro)
  - Next Track (Macro)
- Productivity (Category)
  - Code Snippets (Sub-category)
    - Console.WriteLine (Macro)
    - Public Class (Macro)

### 2. The AutoHotkey (AHK) Backend
- A script to hook `Volume_Up`, `Volume_Down`, and `Volume_Mute`.
- Inter-Process Communication (IPC) logic. We can use standard input/output (running the C# app as a child process) or a local port (UDP/TCP) to send simple text commands.
- Macro execution logic mapping IDs to actual keyboard sends.

### 3. The C# UI Frontend
- A borderless, transparent, topmost window.
- Drawing a circular wheel with "slices" corresponding to the current menu items.
- IPC receiver to listen to AHK commands (Show, Hide, Next, Prev, Select).
- IPC sender to send execution commands back to AHK.

## Open Questions to Decide On
1. **Back Navigation:** Since the knob can only rotate and be pressed, how does the user navigate "Back" to a previous menu level? (e.g., A dedicated "Back" slice in every menu, long-pressing the knob, or a separate keyboard key?)
2. **Toggle Mode:** Should the knob *always* act as a macro menu, or should there be a hotkey to toggle between "Volume Mode" and "Macro Mode"?
3. **Visual Aesthetics:** Do you want icons, text, or specific colors for the UI slices?
